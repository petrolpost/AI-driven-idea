# AI辅助技术文章分析及项目开发规则

> 本规则文档定义了从技术文章中提取工具项目的标准化流程、项目组织结构和开发最佳实践。

## 📖 技术文章分析规则

### 1. 文章阅读与分析

**阅读策略**:

- 🎯 **目标导向**: 重点关注可实现的技术方案、决策框架、工具方法
- 🔍 **深度理解**: 分析文章的核心观点、技术原理、应用场景
- 💡 **价值提取**: 识别具有实用价值的概念、算法、评估方法
- 🚀 **可行性评估**: 判断技术方案的实现难度和实用性

**分析维度**:

- **技术深度**: 是否包含具体的技术实现细节
- **实用价值**: 能否解决实际问题或提升工作效率
- **创新性**: 是否提供新的思路或改进现有方案
- **可扩展性**: 是否具备进一步开发和优化的潜力

### 2. 工具项目识别标准

**合格项目特征**:

- ✅ **明确需求**: 有清晰的问题定义和解决目标
- ✅ **技术可行**: 基于现有技术栈可以实现
- ✅ **实用价值**: 能够提供实际的业务或开发价值
- ✅ **独立性**: 可以作为独立项目进行开发和维护

**项目类型分类**:

- 🔧 **工具类**: 提升开发效率的实用工具
- 📊 **分析类**: 数据分析、决策支持系统
- 🤖 **AI应用**: 基于AI技术的智能应用
- 🌐 **Web服务**: 提供在线服务的Web应用
- 📚 **框架类**: 可复用的开发框架或库

## 🏗️ 项目组织结构规则

### 1. 目录结构标准

```
项目根目录/
├── 项目导航.md                 # 项目索引文件
├── .trae/
│   └── rules/
│       └── project_rules.md    # 本规则文件
└── [项目名称]/                 # 具体项目目录
    ├── README.md               # 项目说明文档
    ├── pyproject.toml          # Python项目配置
    ├── .python-version         # Python版本指定
    ├── Makefile               # 开发命令集合
    ├── .gitignore             # Git忽略规则
    ├── .env.example           # 环境变量示例
    ├── main.py                # 主程序入口
    ├── run.py                 # 运行脚本
    ├── docs/                  # 文档目录
    ├── examples/              # 示例代码
    ├── tests/                 # 测试代码
    ├── static/                # 静态资源
    └── assets/                # 项目资源
```

### 2. 命名规范

**项目命名**:

- 使用小写字母和连字符: `ai-decision-framework`
- 体现项目核心功能: `text-analyzer`, `api-gateway`
- 避免过长或过于抽象的名称

**文件命名**:

- Python文件: 小写字母+下划线 `decision_engine.py`
- 文档文件: 小写字母+连字符 `case-studies.md`
- 配置文件: 标准名称 `pyproject.toml`, `Makefile`

### 3. 项目导航维护

**更新时机**:

- ✅ 新项目创建时立即更新
- 🔄 项目状态变更时及时同步
- 📊 定期检查和维护统计信息

**更新内容**:

- 项目概览表格
- 详细项目信息
- 统计数据
- 更新日志

**项目链接规范**:

- 📎 **项目名称链接**: 项目概览表格中的项目名称必须链接到对应项目目录下的 `README.md`文件
- 🔗 **链接格式**: 使用相对路径格式 `[项目名称](项目目录/README.md)`
- ✅ **链接验证**: 确保所有项目链接指向正确的README文件路径
- 🔄 **链接维护**: 项目目录变更时同步更新导航文件中的链接

## 🛠️ 技术栈规范

### 1. Python项目标准

**包管理器**: 优先使用 `uv`

- 快速依赖解析和安装
- 现代化的项目管理
- 与传统pip兼容

**项目配置**: 使用 `pyproject.toml`

- 统一的项目元数据
- 依赖管理配置
- 工具配置集中化

**开发工具链**:

- 🎨 **代码格式化**: Black
- 🔍 **代码检查**: Ruff + MyPy
- 🧪 **测试框架**: pytest
- 📊 **覆盖率**: coverage
- 🔧 **构建工具**: Makefile

### 2. Web应用标准

**后端框架**: FastAPI

- 现代化的API开发
- 自动文档生成
- 高性能异步支持

**前端技术**: 原生HTML/CSS/JS

- 轻量级实现
- 快速原型开发
- 易于维护和扩展

**部署方式**: Uvicorn

- ASGI服务器
- 开发和生产环境支持
- 热重载功能

## 📋 开发流程规范

**步骤清单**:

1. 📁 创建项目目录结构
2. 📝 编写项目配置文件
3. 🔧 设置开发工具链
4. 📚 创建基础文档

- ✅ 代码质量检查通过
- ✅ 服务正常启动

### 2. 开发阶段管理

**阶段定义**:

- 📋 **计划中**: 项目已规划，待开始开发
- 🚧 **进行中**: 项目正在开发中
- ✅ **完成**: 项目开发完成，功能稳定可用
- ⏸️ **暂停**: 项目暂时停止开发
- ❌ **已废弃**: 项目不再维护

**状态更新**:

- 及时更新项目导航文件
- 记录重要里程碑
- 维护更新日志

### 3. 代码质量标准

**代码规范**:

- 遵循PEP 8编码规范
- 使用类型注解
- 编写清晰的文档字符串
- 保持函数和类的单一职责

**测试要求**:

- 单元测试覆盖率 > 80%
- 集成测试覆盖主要功能
- 性能测试（如适用）
- 错误处理测试

**文档标准**:

- README.md包含完整的使用说明
- API文档自动生成
- 提供使用示例
- 维护更新日志
